Descripción
El proyecto consiste en el desarrollo de una plataforma web integral diseñada para la gestión 
inteligente de inventarios 
en una cafetería. A diferencia de un sistema de almacén convencional, esta solución vincula directamente la salida de productos finalizados con el stock de materia prima. 
Mediante la digitalización de recetas estándar, el sistema realiza un descuento automático y proporcional de los insumos en el momento en que se registra la venta de un producto, permitiendo un control de existencias y en tiempo real sin necesidad de intervención manual constante. 
Adicionalmente, el sistema incorporará una pasarela de pagos integrada que permitirá procesar transacciones digitales en el punto de venta, aceptando pagos con billeteras digitales. Esta funcionalidad ampliará las opciones de cobro para la cafetería “Dante”, reducirá el manejo de efectivo y facilitará la conciliación automática de las ventas registradas con los movimientos de insumos en el inventario
.
 Objetivos
Objetivo General 
Desarrollar un sistema web de gestión inteligente de inventarios integrado con una pasarela de pagos mediante PayPal para optimizar el control de productos, agilizar las ventas y mejorar la administración operativa de la cafetería “Dante”.
Objetivo Específicos
Desarrollar un panel de control que permita visualizar información relevante sobre ventas, inventario y productos más vendidos para apoyar la toma de decisiones en la cafetería “Dante”.
Diseñar un sistema de gestión de productos que facilite el registro, actualización y control de los productos ofrecidos por la cafetería.
Implementar un módulo de control de inventario que permita registrar y monitorear las entradas y salidas de productos e insumos en tiempo real.
Desarrollar un sistema de control de insumos basado en el método Kardex para medir y administrar el consumo de materias primas utilizadas en la preparación de productos.
Implementar un módulo de ventas que permita registrar transacciones, generar comprobantes y actualizar automáticamente el inventario según las ventas realizadas.
Integrar una pasarela de pagos mediante PayPal para facilitar transacciones seguras y rápidas dentro del sistema web.
Diseñar un sistema de autenticación y gestión de usuarios que permita controlar el acceso al sistema según los roles establecidos.

Análisis de la Situación actual
Planteamiento del problema
En la actualidad, ‘Dante’ carece de un sistema integral que permita realizar el inventario y control de insumos con actualizaciones en tiempo real, adaptado a sus necesidades y dinámicas operativas específicas. Esta inexistencia tecnológica genera una limitación crítica para la gestión de recursos, pues no se puede conocer de manera precisa y oportuna el consumo real de los insumos que abastecen los procesos de ‘Dante’.
La falta de un sistema con capacidades de actualización inmediata impide registrar las entradas, salidas y movimientos de insumos al momento en que ocurren, lo que provoca desfases entre la información disponible y la realidad del almacén o puntos de uso. Esta situación se agrava porque tampoco existe una herramienta que permita generar reportes dinámicos y visuales que resalten la información más relevante sobre el comportamiento del consumo, como patrones de uso, niveles críticos de stock, insumos de mayor rotación o aquellos con alto índice de pérdida.
Como consecuencia directa de esta problemática, ‘Dante’ enfrenta múltiples afectaciones operativas y económicas, entre las que destacan:
Pérdidas de insumos por vencimiento, obsolescencia, extravío o mal manejo, al no existir trazabilidad ni alertas tempranas.
Stock en demasía acumulado innecesariamente, lo que incrementa los costos de almacenamiento, inmoviliza capital y genera ocupación ineficiente de espacios.
Falta de insumos críticos en momentos clave (como consecuencia indirecta de no tener visibilidad real), lo que puede paralizar actividades o forzar compras de emergencia.
Toma de decisiones reactiva y poco fundamentada, basada en datos desactualizados o incompletos, lo que perpetúa ciclos de sobrecompra o desabasto.
Dificultad para planificar compras y presupuestos, al no contar con históricos confiables ni tendencias de consumo reales.
De esta manera, la ausencia de un sistema de inventario y control en tiempo real adaptado a los requerimientos de ‘Dante’ no solo limita el conocimiento objetivo del consumo de insumos, sino que genera ineficiencias operativas progresivas que impactan directamente en la rentabilidad, la continuidad de los procesos y la capacidad de respuesta de la organización.
Consideraciones de hardware y software
Para la implementación del sistema se requieren los siguientes recursos de hardware y software:
Hardware requerido:
• Computadoras de desarrollo: procesador Intel Core i5 o superior, 8 GB de RAM mínimo, almacenamiento SSD de 256 GB, sistema operativo Windows 10/11 o Linux Ubuntu 22.04.
• Servidor VPS en la nube (para producción): 2 vCPU, 4 GB RAM, 50 GB SSD, con soporte para PHP 8.x y MySQL.
• Dispositivos de acceso en la cafetería: computadora de escritorio o tablet con navegador web moderno (Chrome, Firefox o Edge) y conexión a internet estable.
Software requerido:
Frontend: Next.js 15.x (corriendo sobre Node.js 24.x o superior). Al ser un framework de React, se lleva de maravilla con interfaces dinámicas y un consumo de API fluido.
Backend API: Laravel 11.x (corriendo sobre PHP 8.3 u 8.4). Es la versión actual más robusta para construir APIs RESTful estructuradas de forma rápida.
Base de Datos: MySQL 8.4 LTS o MySQL 9. Es ideal para el manejo relacional del sistema Kardex que planeas implementar.
Entorno de desarrollo: Visual Studio Code 2026  con extensiones para PHP.
Control de versiones: GitHub 2.4 con flujo de trabajo por ramas (GitFlow).
• Pasarela de pagos: API de Paypal, con soporte para tarjetas de crédito, débito y billeteras digitales.
• Infraestructura de red: conexión a internet de banda ancha (mínimo 10 Mbps), dominio web (.com) y certificado SSL/TLS para comunicaciones seguras (HTTPS).
• Navegadores compatibles: Google Chrome 110+, Mozilla Firefox 110+ o Microsoft Edge 110+ para el acceso al sistema desde los dispositivos de la cafetería.

Factibilidad Operativa
El sistema es operativamente viable y generará un impacto positivo directo en los procesos de la cafetería "Dante":
Permite llevar un registro preciso y en tiempo real del inventario de insumos, eliminando los registros manuales propensos a errores humanos.
Genera un kardex automatizado que relaciona cada venta con el consumo teórico de insumos, permitiendo detectar discrepancias con el consumo real.
Emite alertas automáticas de stock mínimo para prevenir desabastecimiento en horas de operación.
Reduce el tiempo dedicado al control manual del inventario en aproximadamente 3 horas semanales, que el administrador puede redirigir a actividades de mayor valor.
Mejora la comunicación con proveedores al centralizar el historial de compras y movimientos de stock en un solo sistema.
Puede ser operado por personal con conocimientos básicos en informática, gracias a una interfaz intuitiva diseñada con Tailwind CSS.

Factibilidad Social
Aceptación del Sistema
El sistema propuesto presenta alta aceptación por parte de los stakeholders identificados, dado que responde directamente a una necesidad operativa concreta: la gestión eficiente del inventario. La interfaz será diseñada para ser intuitiva y accesible para personal con conocimientos tecnológicos básicos, reduciendo la curva de aprendizaje. Este enfoque contribuye al ODS 9: Industria, innovación e infraestructura, al promover la digitalización de procesos en pequeños negocios mediante soluciones tecnológicas accesibles.
Impacto Social
El sistema contribuye a la formalización y digitalización de los procesos internos de la cafetería “Dante”, promoviendo una cultura de orden, transparencia y toma de decisiones basada en datos reales. Esto repercute positivamente en la sostenibilidad económica del negocio y en la calidad del servicio ofrecido a los clientes, alineándose con el ODS 8: Trabajo decente y crecimiento económico, al mejorar la eficiencia operativa, la organización interna y la productividad del negocio.
Aspectos Éticos
El proyecto se desarrolla bajo principios de transparencia y responsabilidad. El sistema centraliza y protege la información del negocio, fomenta prácticas de gestión ordenadas y evita la manipulación arbitraria del inventario mediante el registro auditable de cada movimiento de stock. Esto se relaciona con el ODS 16: Paz, justicia e instituciones sólidas, al fortalecer la confianza, la trazabilidad de la información y la correcta gestión de los recursos internos.
Factibilidad Ambiental
El proyecto no genera contaminación directa ni residuos físicos, al tratarse de una aplicación de software. Su funcionamiento en entornos digitales elimina la necesidad de registros en papel, reduciendo indirectamente el impacto ambiental generado por la gestión manual del inventario. El consumo de recursos se limita al uso de equipos de cómputo y energía eléctrica, propio de cualquier actividad informática de este tipo. La implementación en un VPS en la nube optimiza el consumo energético al evitar la operación de servidores físicos locales. En términos generales, el sistema contribuye a prácticas de negocio más sostenibles al reemplazar procesos físicos por procesos digitales eficientes, en concordancia con el ODS 12: Producción y consumo responsables y el ODS 13: Acción por el clima.

Viabilidad Económica Robusta: Los tres indicadores financieros calculados con el nuevo esquema de costos (B/C = 2.43, VAN = S/ 10,580.07 y TIR = 84.77%) confirman de manera categórica que el proyecto EVA es altamente rentable, genera un valor neto positivo para la Cafetería "Dante" y su ejecución técnica está plenamente justificada.
Consolidación del Análisis Financiero: Se ha estructurado y completado íntegramente la sección de Análisis Financiero del documento, subsanando las omisiones del planteamiento original mediante la proyección rigurosa de flujos de caja anualizados reales y la aplicación formal de criterios de evaluación de proyectos de TI.
Modernización y Sinceramiento del Stack: La adopción de una arquitectura desacoplada basada en Next.js 15.x (Frontend), Laravel 11.x (Backend API) y MySQL 8.4 LTS (Base de datos) dota al sistema de altos estándares de escalabilidad y rendimiento. Asimismo, la inclusión de un presupuesto de S/ 600.00 anuales para soporte técnico preventivo sincera la operación cloud del software, mitigando riesgos de depreciación o abandono tecnológico.
Absorción de Observaciones Críticas: Se resolvieron con éxito las inconsistencias estructurales del presupuesto inicial. Se eliminó la ambigüedad de la nota "por integrante × 5" en los costos operativos —fijando sólidamente la inversión en S/ 7,375.00— y se normalizó la tabla del entorno, separando con claridad los conceptos de renovación anual de dominio frente al hosting mensualizado.
Retorno Técnico de la Inversión (Payback): El análisis del flujo neto anualizado demuestra que el capital inicial queda comprometido al cierre de las 12 semanas (3 meses comerciales) de la fase de desarrollo. Gracias a los ahorros generados en el control de mermas y tiempos, la inversión se recupera de forma íntegra a los 12.25 meses desde su puesta en producción (equivalente al mes 15.25 contados desde el día uno del cronograma de trabajo).
